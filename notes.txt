Anotacoes:
- Sail nao existia porque o vendor ainda nao foi gerado; instalar via Docker com imagem laravelsail/php82-composer para rodar:
  docker run --rm \
    -u "$(id -u):$(id -g)" \
    -v "$(pwd)":/var/www/html \
    -w /var/www/html \
    laravelsail/php82-composer:latest \
    composer install

- Subir containers e instalar dependencias JS pelo Sail:

  ./vendor/bin/sail up -d
  ./vendor/bin/sail npm install

- Erro de sessions: rodar migrate ou trocar driver:

  ./vendor/bin/sail artisan migrate
  SESSION_DRIVER=file
  ./vendor/bin/sail artisan config:clear

- Nwidart Modules (instalacao e criacao):

  ./vendor/bin/sail composer require nwidart/laravel-modules
  ./vendor/bin/sail artisan vendor:publish --provider="Nwidart\Modules\LaravelModulesServiceProvider"
  ./vendor/bin/sail artisan module:make Blog

- Correcao do erro "BlogServiceProvider not found":

  Ajuste de autoload no composer.json para incluir classmap de Modules/ e remover o PSR-4 direto.
  Depois rodar:

  ./vendor/bin/sail composer dump-autoload
  ./vendor/bin/sail artisan config:clear

- Detalhe do ajuste no composer.json (antes/depois):

  Antes (problematico):
  "psr-4": { "Modules\\": "Modules/" }

  Depois (funciona):
  "psr-4": {
    "App\\": "app/",
    "Database\\Factories\\": "database/factories/",
    "Database\\Seeders\\": "database/seeders/"
  },
  "classmap": ["Modules/"]

- Correcao de timezone (hora errada):

  config/app.php -> usar APP_TIMEZONE:
  'timezone' => env('APP_TIMEZONE', 'UTC'),

  .env -> definir:
  APP_TIMEZONE=America/Sao_Paulo

  Aplicar:
  ./vendor/bin/sail artisan config:clear
